# Galio Theme usage
All colors, sizes and layout tokens live in the library's `src/theme` folder. Modern Galio exposes a simple, semantic theme API that is easier to customize and adapts automatically to light/dark mode.

<CarbonAds serve="CE7IP3L" placement="galioio" />

## Goals of the modern theme

- Provide a single source of truth for semantic tokens (`colors`, `sizes`, `shadows`).
- Let applications override only the tokens they need (deep-merged).
- Offer simple hooks: `useTheme()` and `useColors()` for component code.
- Keep backward compatibility with legacy exports while encouraging migration.

## Quick start

1. Install Galio normally.
2. Wrap your app with `GalioProvider` and pass minimal overrides (your brand colors).

```jsx
import { GalioProvider } from 'galio-framework';
import { myAppTheme } from './theme';

export default function App() {
  return (
    <GalioProvider theme={myAppTheme}>
      <YourApp />
    </GalioProvider>
  );
}
```

Where `myAppTheme` contains only what you want to override, for example:

```js
export const myAppTheme = {
  colors: {
    primary: '#6366F1',
    success: '#10B981',
    error: '#EF4444',
  },
  sizes: {
    BASE: 18,
    BUTTON_HEIGHT: 48,
  }
};
```

## Core APIs

### `GalioProvider`

- Props: `mode?: 'light' | 'dark' | 'auto'`, `theme?: { colors?, sizes?, shadows?, COLORS?, SIZES?, customTheme? }`.
- Behavior: deep-merges provided tokens with library defaults and exposes the resulting theme via context.

### `useTheme()`

Returns the full theme object: `{ colors, sizes, shadows, mode, ... }`.
Use when you need multiple theme sections (colors + sizes).

```js
const { colors, sizes, mode } = useTheme();
```

### `useColors()`

Returns just the semantic `colors` object for the current mode. Use this inside components for concise color access.

```js
const colors = useColors();
return <View style={{ backgroundColor: colors.background }} />;
```

## What tokens exist (summary)

The library exposes these modern token groups (defaults are in `src/theme`):

- `colors` — semantic color tokens for backgrounds, text, states (primary, success, error, warning, info), surfaces, borders, disabled colors and utility colors like `white`/`black`/`transparent`.
- `sizes` — base sizing constants and component-specific sizes (ICON, BUTTON_HEIGHT, CARD_* values, CHECKBOX sizes, etc.).
- `shadows` — named shadow presets (`xs`, `sm`, `md`, `lg`, `xl`, plus legacy keys) with platform-specific values for `ios`, `android`, and `web`.

Refer to `src/theme/colors.tsx` and `src/theme/sizes.tsx` for the full token lists.

## Compact reference — Most-used tokens

### Colors (common)

| Token | Purpose |
|-------|---------|
| `background` | App background color |
| `surface` | Card / panel background |
| `text` | Primary text color |
| `textSecondary` | Secondary / muted text |
| `primary` | Brand primary color |
| `primaryHover` | Primary hover/active state |
| `success` | Success state color |
| `error` | Error state color |
| `warning` | Warning state color |
| `info` | Informational color |
| `border` | Standard border color |
| `disabled` | Disabled state color |

### Sizes (common)

| Token | Purpose |
|-------|---------|
| `BASE` | Base unit for spacing and typography |
| `FONT` | Default font size |
| `ICON` | Default icon size |
| `ICON_MEDIUM` | Medium icon size |
| `ICON_LARGE` | Large icon size |
| `BUTTON_HEIGHT` | Standard button height |
| `CARD_WIDTH` | Default card width |
| `CARD_BORDER_RADIUS` | Card corner radius |
| `CHECKBOX_WIDTH` | Checkbox size |

Refer to `src/theme/colors.tsx` and `src/theme/sizes.tsx` for the full list and exact defaults.

## Theming examples

Use semantic tokens directly in components:

```js
import { useColors } from 'galio-framework';

function MyCard() {
  const colors = useColors();
  return (
    <View style={{ backgroundColor: colors.surface, borderColor: colors.border }}>
      <Text style={{ color: colors.text }}>Hello</Text>
    </View>
  );
}
```

Override shadows and sizes in the provider if you need a different scale or shadow system:

```js
<GalioProvider theme={{
  shadows: { md: { ios: {...}, android: {...}, web: {...} } },
  sizes: { BASE: 18 }
}}>
  <App />
</GalioProvider>
```


## How to add a custom theme

Follow these minimal steps to add a custom theme to your app. The library deep-merges your overrides, so you only need to supply the tokens you want to change.

1. Create a `src/theme.ts` (or `app/theme.ts`) file in your app. Use the modern token names (`colors`, `sizes`, `shadows`). Example (TypeScript):

```ts
import type { Partial } from 'utility-types';
import type { GalioTheme } from 'galio-framework/src/theme';

export const myAppTheme: Partial<Pick<GalioTheme, 'colors' | 'sizes' | 'shadows'>> = {
  colors: {
    primary: '#6366F1',
    primaryHover: '#4F46E5',
    background: '#FAFAFA',
    text: '#111827',
  },
  sizes: {
    BASE: 18,
    BUTTON_HEIGHT: 48,
  },
  // Optional: override shadow presets
  shadows: {
    md: {
      ios: { shadowColor: '#000', shadowOffset: { width: 0, height: 6 }, shadowOpacity: 0.12, shadowRadius: 8 },
      android: { elevation: 6 },
      web: { boxShadow: '0px 6px 12px rgba(0,0,0,0.12)' },
    }
  }
};
```

2. Provide it to `GalioProvider` at the root of your app:

```tsx
import { GalioProvider } from 'galio-framework';
import { myAppTheme } from './theme';

export default function App() {
  return (
    <GalioProvider theme={myAppTheme}>
      <YourApp />
    </GalioProvider>
  );
}
```

3. Use tokens via `useColors()` or `useTheme()` in components. Example:

```tsx
import { useColors } from 'galio-framework';

function Header() {
  const colors = useColors();
  return <View style={{ backgroundColor: colors.primary }} />;
}
```


## Migration notes (short)

- Legacy keys (`COLORS`, `SIZES`, `LIGHT_MODE`, `DARK_MODE`) are still exported for compatibility but are deprecated. Prefer `colors`, `sizes`, and `shadows`.
- Use `useColors()` instead of `useThemeColors()` and `useTheme()` instead of `useGalioTheme()`.
- `GalioProvider` accepts both new and legacy theme shapes and will warn when legacy keys are detected.

## Developer tips

- Override only brand tokens — the provider deep-merges your values with safe defaults.
- Prefer semantic color names (`primary`, `success`, `error`) rather than raw hex strings across your app for consistent light/dark behavior.
- For responsive `sizes`, compute `cardWidth` / `buttonWidth` in your app and pass them via `theme.sizes` or directly as props to components.
